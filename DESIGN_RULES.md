# DESIGN_RULES.md

## 목적
UpLog의 디자인 일관성을 유지하고, 월간 캘린더 사용성을 최우선으로 보장합니다.

## 1) 제품 경험 원칙
1. **Calendar First**: 모든 화면은 캘린더 탐색 흐름을 방해하지 않는다.
2. **Fast Recognition**: 사용자가 날짜 상태를 즉시 인식할 수 있어야 한다.
3. **Low Cognitive Load**: 한 화면에서 필요한 정보만 보여준다.
4. **Progressive Disclosure**: 고급 정보는 단계적으로 노출한다.

## 2) 레이아웃 규칙
- 기본 레이아웃: 상단 네비게이션 + 월간 캘린더 본문 + 우측/하단 상세 패널
- 월간 그리드는 항상 중심 영역을 차지
- 날짜 셀 높이/간격은 일정 개수 변동에도 안정적으로 유지

## 3) 타이포그래피/컬러
- 가독성 우선, 장식적 요소 최소화
- 날짜 숫자/오늘/선택/이벤트 상태는 명확히 대비
- 컬러는 의미 기반으로 제한 사용 (예: 기본/강조/경고)

## 4) 인터랙션 규칙
- 월 이동 버튼은 항상 동일 위치
- 날짜 클릭 시 선택 상태가 즉시 반영
- 로딩/빈 상태/오류 상태를 명시적으로 표시

## 5) 접근성 규칙
- 키보드 탐색 지원 (월 이동, 날짜 포커스, 일정 목록 접근)
- 텍스트 대비 준수
- 스크린리더용 레이블 제공

## 6) 범위 제약
초기 단계에서는 아래를 디자인 범위에서 제외합니다.
- notes 전용 복합 편집 UI
- todo 전용 보드 UI
- AI 기반 제안 패널
