# BIGDATA_UI_RULES.md

## 목적
월간 캘린더에서 데이터가 많아져도 UpLog UI가 무너지지 않도록 규칙을 정의합니다.

## 1) 기본 원칙
1. **Grid Stability**: 이벤트 수가 많아도 월간 그리드 구조는 유지한다.
2. **Summary over Detail**: 셀 내부에는 요약만, 상세는 별도 패널에서 노출한다.
3. **Predictable Overflow**: 넘침 처리 방식은 일관되게 유지한다.

## 2) 월간 셀 데이터 표시 규칙
- 셀 내부에는 최대 N개(예: 2~3개) 이벤트만 직접 표시
- 초과분은 `+N more` 형태로 요약
- `+N more` 클릭 시 해당 날짜 상세 패널/모달로 이동

## 3) 성능 규칙
- 월 단위 렌더링 비용 최소화
- 날짜 셀 렌더링은 메모이제이션/가상화 전략 검토
- 대량 데이터 로드시 스켈레톤 또는 단계적 로딩 사용

## 4) 정보 계층 규칙
- 1계층: 월간 셀 요약
- 2계층: 선택 날짜 상세 리스트
- 3계층: 개별 일정 상세/편집

## 5) 상태 표시 규칙
- 로딩 상태: 셀 전체 대신 영역 단위로 표시
- 오류 상태: 사용자 행동 가능한 재시도 제공
- 빈 상태: “아직 일정이 없습니다” 메시지와 생성 액션 제공

## 6) 비기능 요구
- 월간 전환 시 체감 지연 최소화
- 데이터 1,000+ 건 수준에서도 인터랙션 지연을 줄이는 구조 우선
- 구현 초기에 측정 지표(FPS, 렌더 시간) 수집 포인트 마련
