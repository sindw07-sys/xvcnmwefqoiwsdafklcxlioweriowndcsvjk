# PROJECT_GUIDE.md

## 1) 프로젝트 개요
UpLog는 일정 기록을 중심으로 한 캘린더 서비스입니다. 초기 목표는 월간 캘린더 기반 UX를 완성하는 것입니다.

### 고정 원칙
- 앱 표시 이름은 **UpLog**
- 프로젝트 폴더명은 **uplog**
- 개발은 **웹앱 우선**
- 제품 중심은 **캘린더**
- notes / todo / AI 기능은 **후순위**

## 2) 1차 개발 범위 (MVP)
초기 MVP는 다음 기능만 포함합니다.

### 포함
1. 월간 캘린더 화면
2. 월 이동 (이전 달/다음 달/오늘)
3. 날짜 선택 상태 표시
4. 날짜별 일정 목록 노출(기본형)
5. 일정 생성/수정/삭제의 최소 플로우

### 제외 (보류)
1. Notes 모듈
2. Todo 모듈
3. AI 추천/요약/자동분류
4. 복잡한 통계 대시보드

## 3) 웹 우선 전략
- 반응형 웹 기준으로 UI/UX 설계
- 데스크톱에서 먼저 완성한 뒤 모바일 반응형으로 확장
- PWA 적용은 초기 캘린더 안정화 이후 검토

## 4) 정보 구조
초기 정보 구조는 단순하게 유지합니다.
- Calendar (월간 뷰 중심)
  - Month Grid
  - Selected Day Panel
  - Event CRUD

## 5) 인수인계 체크포인트
- 기능 추가 전 반드시 `DESIGN_RULES.md`와 `BIGDATA_UI_RULES.md`를 확인
- 범위 확장 요청이 와도 MVP 범위(월간 캘린더 중심) 우선
- 보류 기능(notes/todo/AI)은 별도 릴리즈에서 재평가
